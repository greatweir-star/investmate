# InvestMate Repository Spec

> 本文档定义 InvestMate 仓库的组织方式、语言规范、开发协作方式和第一阶段交付边界。

## 1. 项目定位

InvestMate 是一个面向个人投资者的 AI 投研辅助产品。

第一阶段目标不是自动交易，也不是替用户做最终投资判断，而是帮助用户在研究市场、理解持仓、评估风险和形成个人投资计划时，获得一个结构化、可解释、可复盘的 AI 辅助视角。

## 2. 仓库语言规范

- 产品文档：中文。
- Prompt 文档：中文为主，必要时保留英文变量名。
- 代码文件名：英文。
- API 路径：英文。
- 数据库表名和字段名：英文。
- 用户界面文案：中文。

## 4. 第一阶段产品边界

第一阶段只做投研辅助和决策支持，不做：

- 自动下单；
- 账户托管；
- 收益承诺；
- 高频交易；
- 盘中即时交易指令；
- 面向用户的保证性结论。

## 5. 第一阶段核心模块

### 5.1 市场状态模块

基于指数、行业、成交、波动和市场宽度等信息，形成市场环境概览。

### 5.2 标的分析模块

围绕单个股票、ETF 或指数进行结构化分析，包括趋势、估值、行业、波动、流动性和风险因素。

### 5.3 持仓分析模块

根据用户手动录入的持仓，识别集中度、行业暴露、风格偏差、波动风险和组合结构问题。

### 5.4 AI 解释模块

将结构化数据和规则判断转化为用户可理解的中文解释。

### 5.5 历史记录模块

记录系统每一次输出，方便后续追踪、复盘和产品迭代。

## 6. AI Native 开发原则

1. 先写规格，再写代码。
2. Prompt 和规则都需要版本化管理。
3. 每个 AI 输出必须有结构化 JSON 结果，不只生成自然语言。
4. 每个模块都要清晰定义输入、输出、边界和异常情况。
5. 每个页面都要回答一个明确的用户问题。

## 7. MVP 技术建议

- 前端：Next.js 或 React。
- 后端：FastAPI 或 Next.js API Routes。
- 数据库：PostgreSQL。
- 数据源：Tushare Pro。
- AI 层：结构化 Prompt + JSON 输出。
- 部署：Vercel、Railway、Render 或单机 Docker。

## 8. Codex 使用方式

Codex 应优先阅读以下文件：

1. `README.md`
2. `REPOSITORY_SPEC.md`
3. `docs/MVP产品设计.md`
4. `docs/决策引擎.md`
5. `specs/API设计.md`
6. `specs/数据库设计.md`
7. `tasks/第一阶段开发计划.md`

Codex 开发时应遵守：

- 不擅自扩大产品范围；
- 不引入自动交易能力；
- 不把自然语言输出作为唯一结果；
- 所有核心分析结果必须有结构化数据；
- 保持产品中文体验。
