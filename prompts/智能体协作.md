# 智能体协作

## 1. 目标

本文档定义 InvestMate MVP 中各个 Agent 的协作方式。

第一阶段不追求复杂多智能体自治，而是采用清晰、可控、可调试的流水线式协作。

## 2. Agent 列表

MVP 包含四类 Agent：

1. 市场分析 Agent；
2. 标的分析 Agent；
3. 持仓分析 Agent；
4. 复盘 Agent。

## 3. 协作原则

1. 数据和规则先行，AI 解释后置。
2. 每个 Agent 只处理自己的任务。
3. 所有 Agent 输出必须是结构化 JSON。
4. Agent 之间不直接相互调用，由后端服务编排。
5. 每次输出必须保存 Prompt 版本和模型版本。

## 4. 标的分析链路

```text
用户输入标的
  ↓
后端查询标的和行情数据
  ↓
信号服务计算基础指标
  ↓
标的分析 Agent 生成结构化解释
  ↓
分析记录写入数据库
  ↓
前端展示
```

## 5. 持仓分析链路

```text
用户录入持仓
  ↓
后端补全标的信息
  ↓
计算组合集中度和风险暴露
  ↓
持仓分析 Agent 生成结构化解释
  ↓
保存组合快照和分析记录
  ↓
前端展示
```

## 6. 市场分析链路

```text
数据同步完成
  ↓
读取指数和市场数据
  ↓
计算市场信号
  ↓
市场分析 Agent 生成市场摘要
  ↓
保存市场状态
  ↓
首页展示
```

## 7. 复盘链路

```text
用户打开历史记录
  ↓
选择某条历史分析
  ↓
系统读取后续行情数据
  ↓
复盘 Agent 对比当时判断和后续变化
  ↓
生成复盘摘要
```

## 8. Agent 输出约束

每个 Agent 输出必须包含：

- analysis_type；
- summary；
- confidence；
- risk_level，如适用；
- key_factors；
- risk_factors；
- explanation。

## 9. 编排服务

后端应创建 `analysis_service` 作为统一编排层。

职责：

1. 接收 API 请求；
2. 准备数据；
3. 调用信号服务；
4. 调用对应 Agent；
5. 校验输出结构；
6. 保存历史记录；
7. 返回前端。

## 10. 后续扩展

未来可以增加：

- 新闻解读 Agent；
- 公告解读 Agent；
- 财报解读 Agent；
- 行业分析 Agent；
- 内容生成 Agent。

但任何新增 Agent 都必须先定义输入、输出和边界。
