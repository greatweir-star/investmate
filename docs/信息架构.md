# 信息架构

## 1. 目标

本文档定义 InvestMate MVP 的页面结构、信息层级和主要入口。

第一阶段的产品不追求功能复杂，而是让用户快速完成三件事：

1. 了解当前市场环境；
2. 查看某个标的的结构化分析；
3. 查看自己持仓结构中的主要问题。

## 2. 顶层导航

MVP 顶层导航包含五个入口：

```text
首页
标的分析
持仓分析
历史记录
设置
```

## 3. 页面结构

### 3.1 首页

首页回答：今天我应该先关注什么？

模块：

- 市场状态卡片；
- 主要风险卡片；
- 关注方向卡片；
- 快速分析入口；
- 最近历史记录。

### 3.2 标的分析

标的分析页回答：这个标的当前有哪些值得关注的因素？

模块：

- 标的搜索框；
- 基础信息区；
- 评分区；
- 关键因素；
- 风险因素；
- 中文解释；
- 数据时间。

### 3.3 持仓分析

持仓分析页回答：我的组合结构有什么明显问题？

模块：

- 持仓录入表格；
- 单一标的集中度；
- 行业暴露；
- 风格暴露；
- 组合风险摘要；
- 用户投资 DNA 匹配度。

### 3.4 历史记录

历史记录页回答：系统过去输出过什么？

模块：

- 分析记录列表；
- 分析类型筛选；
- 时间筛选；
- 详情页入口。

### 3.5 设置

设置页回答：系统如何根据我的情况工作？

模块：

- 数据源配置；
- AI 服务配置；
- 用户投资 DNA；
- 数据更新时间；
- 系统说明。

## 4. MVP 信息优先级

第一优先级：

- 摘要；
- 风险等级；
- 评分；
- 关键因素；
- 风险因素。

第二优先级：

- 详细解释；
- 数据来源；
- 数据日期；
- 历史记录。

第三优先级：

- 图表；
- 高级筛选；
- 多维对比；
- 导出功能。

## 5. 交互原则

1. 首页不要堆砌信息。
2. 分析结果先给摘要，再展开细节。
3. 风险提示必须和结论同屏出现。
4. 数据日期必须可见。
5. 用户不需要理解所有指标，也能读懂摘要。

## 6. 后续扩展

未来可增加：

- 关注列表；
- 周度复盘；
- 组合报告；
- 数据看板；
- 多资产视图。
