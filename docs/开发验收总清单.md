# 开发验收总清单

## 1. 目标

本文档用于验收 InvestMate 第一阶段 MVP 是否达到可演示、可内测、可继续迭代的状态。

## 2. 页面验收

### 首页

- 页面可以访问；
- 可以展示市场状态；
- 可以看到快速入口；
- 可以看到最近记录；
- 空状态有中文提示。

### 标的分析页

- 可以输入标的；
- 可以展示候选结果；
- 可以提交分析；
- 可以展示评分；
- 可以展示因素列表；
- 可以展示中文解释；
- 可以保存历史记录。

### 持仓分析页

- 可以新增持仓；
- 可以编辑持仓；
- 可以删除持仓；
- 可以保存持仓；
- 可以展示组合概览；
- 可以展示中文解释。

### 历史记录页

- 可以展示历史列表；
- 可以查看详情；
- 可以展示输入和输出；
- 空状态有中文提示。

### 设置页

- 可以展示基础配置状态；
- 可以编辑用户偏好；
- 可以保存；
- 不展示敏感字段明文。

## 3. API 验收

所有 API 必须：

- 返回统一结构；
- 成功和失败都有明确结果；
- 用户可见信息使用中文；
- 字段命名使用英文；
- 第一版 mock 数据可用。

## 4. 数据库验收

- 可以创建 MVP 所需表；
- 可以写入 demo 用户；
- 可以保存持仓；
- 可以保存分析记录；
- 可以按时间查询历史记录。

## 5. Agent 验收

- 输出结构化 JSON；
- 包含 summary；
- 包含 confidence；
- 包含 key_factors；
- 包含 explanation；
- 解释基于结构化结果。

## 6. 产品边界验收

- 页面展示数据日期；
- 页面有使用说明；
- 结果页有反馈入口；
- 不出现夸张表达；
- 不替用户做最终判断。

## 7. 内测验收

- 可以邀请用户体验；
- 可以记录反馈；
- 可以整理问题清单；
- 可以形成下版优化项。
