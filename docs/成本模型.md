# 成本模型

## 1. 目标

AI 产品需要控制单次分析成本，避免免费体验阶段成本失控。

本文档用于定义 InvestMate MVP 阶段需要观察的成本项。

## 2. 成本项

### 2.1 模型调用成本

每次分析可能包括：

- 结构化解释生成；
- 报告摘要生成；
- 历史复盘生成。

MVP 阶段应尽量减少模型调用次数。

### 2.2 数据成本

包括：

- 数据源调用；
- 数据同步；
- 数据存储；
- 数据清洗。

### 2.3 服务器成本

包括：

- 前端部署；
- 后端服务；
- 数据库；
- 日志和监控。

### 2.4 人工成本

内测阶段可能需要人工查看反馈、访谈用户和整理问题。

## 3. 控制策略

### 3.1 免费额度

免费用户每天限制分析次数。

### 3.2 Mock 优先

MVP 先用 mock 跑通流程，真实模型和真实数据逐步接入。

### 3.3 缓存结果

同一用户、同一持仓、同一数据日期的报告可以复用。

### 3.4 先规则后 AI

先用规则计算结构化结果，再调用 AI 生成解释。

## 4. 需要记录的指标

- 每次报告模型调用次数；
- 每次报告平均耗时；
- 每次报告平均成本；
- 每日活跃用户数；
- 每日分析次数；
- 用户反馈率。

## 5. 判断标准

如果单次报告成本高于用户可接受价值，需要减少模型调用或调整报告生成方式。
