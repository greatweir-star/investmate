# 价值验证

用于记录 MVP 阶段如何判断产品是否有真实价值。

## 1. 验证目标

验证用户是否愿意使用 InvestMate 完成一次完整流程。

完整流程包括：

```text
进入产品
  ↓
输入标的
  ↓
查看分析结果
  ↓
保存或回看记录
  ↓
提供反馈
```

## 2. 核心问题

需要验证：

1. 用户是否理解产品用途；
2. 用户是否愿意输入标的；
3. 用户是否能看懂结构化结果；
4. 用户是否觉得解释有帮助；
5. 用户是否愿意再次使用。

## 3. 用户分组

### 3.1 首次体验用户

关注点：是否能完成一次基础流程。

### 3.2 深度体验用户

关注点：是否愿意录入更多个人上下文，并查看历史记录。

### 3.3 高意向用户

关注点：是否愿意参与访谈、持续反馈和等待新版本。

## 4. 行为指标

- 首次体验完成率；
- 标的分析完成率；
- 持仓录入完成率；
- 历史记录点击率；
- 反馈提交率；
- 七日内再次访问。

## 5. 访谈指标

访谈中重点观察：

- 用户是否能复述产品价值；
- 用户是否能说出最有用的模块；
- 用户是否愿意推荐给别人；
- 用户是否愿意继续试用。

## 6. 测试动作

### 动作一：首次分析

让用户完成一次标的分析，观察是否能顺利读懂结果。

### 动作二：持仓录入

让用户录入一组模拟或真实组合，观察是否能理解组合摘要。

### 动作三：历史回看

让用户进入历史记录，观察是否能找到过去结果。

### 动作四：反馈提交

让用户在结果页提交一次反馈。

## 7. 通过标准

如果用户愿意重复使用，并主动提出改进建议，说明产品具备继续投入的价值。

如果用户只浏览首页但不继续操作，需要重新检查首页和核心流程。

## 8. 每周输出

每周输出一份验证摘要：

- 完成首次体验人数；
- 完成持仓录入人数；
- 查看历史记录人数；
- 反馈数量；
- 用户原话；
- 下周优化重点。
