# 价值验证

用于记录 MVP 阶段如何判断产品是否有真实价值。

## 1. 验证目标

验证用户是否愿意使用 InvestMate 完成一次完整流程。

完整流程包括：

```text
进入产品
  ↓
输入标的
  ↓
查看分析结果
  ↓
保存或回看记录
  ↓
提供反馈
```

## 2. 核心问题

需要验证：

1. 用户是否理解产品用途；
2. 用户是否愿意输入标的；
3. 用户是否能看懂结构化结果；
4. 用户是否觉得解释有帮助；
5. 用户是否愿意再次使用。

## 3. 行为指标

- 首次体验完成率；
- 标的分析完成率；
- 持仓录入完成率；
- 历史记录点击率；
- 反馈提交率；
- 七日内再次访问。

## 4. 访谈指标

访谈中重点观察：

- 用户是否能复述产品价值；
- 用户是否能说出最有用的模块；
- 用户是否愿意推荐给别人；
- 用户是否愿意继续试用。

## 5. 通过标准

如果用户愿意重复使用，并主动提出改进建议，说明产品具备继续投入的价值。

如果用户只浏览首页但不继续操作，需要重新检查首页和核心流程。
