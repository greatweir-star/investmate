# 决策引擎

## 1. 定位

决策引擎是 InvestMate 的核心模块。

它不负责自动交易，也不直接给出强制性行动指令，而是把市场数据、标的指标、用户投资 DNA 和持仓结构整合起来，生成结构化的投研辅助结果。

## 2. 核心问题

决策引擎回答四类问题：

1. 当前市场环境如何？
2. 某个标的有哪些值得关注的因素？
3. 用户当前组合有哪些结构性风险？
4. 当前分析结果与用户投资 DNA 是否匹配？

## 3. 输入

### 3.1 市场数据

来源：Tushare。

包括：

- 指数日线；
- 个股日线；
- ETF 日线；
- 每日基本面；
- 财务指标；
- 行业分类；
- 交易日历。

### 3.2 用户数据

包括：

- 投资 DNA；
- 手动录入持仓；
- 关注列表；
- 历史分析记录。

### 3.3 系统规则

包括：

- 市场状态规则；
- 标的评分规则；
- 风险识别规则；
- 组合集中度规则；
- 输出合规规则。

## 4. 输出

决策引擎必须输出结构化结果。

示例：

```json
{
  "analysis_type": "asset_review",
  "asset_code": "510300.SH",
  "asset_name": "沪深300ETF",
  "summary": "该标的适合进一步研究，但需要关注市场波动和用户自身仓位情况。",
  "scores": {
    "trend": 72,
    "valuation": 60,
    "liquidity": 85,
    "risk": 40,
    "fit_to_user": 68
  },
  "risk_level": "medium",
  "confidence": 0.74,
  "key_factors": [
    "近期趋势相对稳定",
    "流动性较好",
    "适合作为宽基配置工具"
  ],
  "risk_factors": [
    "若市场整体回撤，仍会受到明显影响",
    "需要结合用户已有权益类仓位评估"
  ],
  "explanation": "从当前数据看，该 ETF 具备较好的流动性和宽基分散特征，但是否适合进一步配置，需要结合用户当前持仓比例和风险承受能力。"
}
```

## 5. 工作流

```text
接收用户问题
  ↓
识别分析类型
  ↓
读取用户投资 DNA
  ↓
获取市场与标的数据
  ↓
计算基础信号
  ↓
执行规则判断
  ↓
生成结构化结果
  ↓
调用 AI 生成中文解释
  ↓
保存历史记录
```

## 6. 信号层

第一阶段只做基础信号，不做复杂机器学习。

### 6.1 趋势信号

- 近 20 日收益；
- 近 60 日收益；
- 价格相对均线位置；
- 均线斜率；
- 回撤幅度。

### 6.2 波动信号

- 近 20 日波动率；
- 近 60 日波动率；
- 最大回撤；
- 单日大幅波动次数。

### 6.3 流动性信号

- 日均成交额；
- 成交额变化；
- 换手率；
- 是否存在流动性不足风险。

### 6.4 估值信号

- PE；
- PB；
- 市值；
- 行业相对估值。

### 6.5 行业信号

- 所属行业近期表现；
- 行业内强弱排名；
- 行业波动；
- 行业集中风险。

## 7. 规则层

### 7.1 市场状态规则

市场状态可以分为：

- 偏弱；
- 震荡；
- 震荡偏强；
- 趋势较强；
- 高波动。

MVP 阶段不需要追求非常精确，只要保持一致、透明、可复盘。

### 7.2 风险识别规则

系统应识别：

- 高波动；
- 流动性不足；
- 单一行业暴露过高；
- 单一标的占比过高；
- 与用户风险偏好不匹配；
- 近期涨幅过快；
- 近期回撤过大。

### 7.3 用户适配规则

同一个分析结果需要结合用户投资 DNA 调整解释。

例如：

- 对保守型用户，强调回撤、波动和集中度；
- 对长期型用户，强调基本面和估值；
- 对短期型用户，强调趋势和流动性；
- 对新手用户，减少术语并解释指标含义。

## 8. AI 解释层

AI 不直接决定结论，而是基于结构化结果生成中文解释。

Prompt 必须包含：

- 输入数据；
- 规则判断；
- 用户投资 DNA；
- 禁止绝对化表达；
- 必须输出风险提示；
- 必须说明不确定性。

## 9. 历史记录

每次输出都需要保存：

- 用户 ID；
- 分析类型；
- 输入参数；
- 结构化结果；
- 中文解释；
- 数据日期；
- 生成时间；
- Prompt 版本；
- 模型版本。

## 10. MVP 验收标准

决策引擎 V0.1 完成标准：

1. 可以分析单个 A 股或 ETF；
2. 可以输出趋势、估值、流动性、风险、适配度评分；
3. 可以结合用户投资 DNA 改变解释侧重点；
4. 可以保存历史结果；
5. 输出中不出现绝对化结论；
6. 结构化结果和中文解释一致。
