# 周度复盘 PRD

## 1. 产品目标

周度复盘用于提高留存，让用户每周回看自己的组合结构和历史变化。

它不是新的复杂功能，而是基于持仓体检报告和历史记录生成周期性总结。

## 2. 用户问题

用户常见问题：

- 我上周看过的组合问题有没有变化；
- 我是否应该重新查看持仓结构；
- 我过去的判断是否值得复盘；
- 我希望有人帮我整理一份简单总结。

## 3. 输入

- user_id；
- 最新持仓；
- 上次报告；
- 本周市场状态；
- 用户投资 DNA；
- 历史反馈。

## 4. 输出模块

周度复盘包含：

1. 本周组合摘要；
2. 与上次报告相比的变化；
3. 需要继续关注的问题；
4. 数据变化说明；
5. 用户反馈回顾；
6. 下周复盘提醒。

## 5. 页面形态

MVP 阶段可以先作为报告页的一种类型。

路径建议：

```text
/reports/weekly/{report_id}
```

也可以先放入历史记录详情中。

## 6. 触发方式

MVP 阶段：用户手动点击生成。

后续阶段：每周固定时间提醒用户生成。

## 7. 留存机制

周度复盘的目标是让用户形成习惯：

```text
周初查看
  ↓
保存记录
  ↓
下周对比
  ↓
形成历史
```

## 8. 验收标准

- 可以基于历史报告生成复盘；
- 可以展示变化摘要；
- 可以保存到历史记录；
- 可以展示数据日期；
- 可以提交反馈。
