# 报告字段规格

## 1. 目标

本文档定义 AI 持仓体检报告的核心字段，方便前端、后端和 Agent 对齐。

## 2. Report

报告主对象字段：

- report_id；
- user_id；
- report_type；
- title；
- summary；
- health_score；
- fit_score；
- data_date；
- generated_at；
- created_at。

## 3. PositionInput

用户输入持仓字段：

- asset_code；
- asset_name；
- weight；
- cost_price；
- note。

## 4. PositionEnriched

系统补全字段：

- asset_code；
- asset_name；
- asset_type；
- industry；
- style；
- weight；
- data_status。

## 5. ScoreItem

评分项字段：

- key；
- name；
- score；
- explanation；
- evidence。

## 6. ExposureItem

分布项字段：

- category；
- name；
- weight；
- description。

## 7. WatchItem

关注事项字段：

- title；
- description；
- evidence；
- level。

## 8. ReportExplanation

中文解释字段：

- short_summary；
- detailed_explanation；
- assumptions；
- data_notes。

## 9. Feedback

反馈字段：

- feedback_id；
- report_id；
- user_id；
- feedback_type；
- feedback_text；
- created_at。

## 10. 前端展示顺序

建议顺序：

1. report title；
2. summary；
3. health_score；
4. score_items；
5. exposure_items；
6. watch_items；
7. explanation；
8. feedback。

## 11. 验收标准

- 后端返回字段稳定；
- 前端可以完整渲染；
- 历史记录可以保存主对象；
- Agent 输出可以映射到字段。
