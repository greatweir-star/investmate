# 信任体系设计

## 1. 目标

投资相关工具的核心不是制造结论，而是让用户知道结论从哪里来。

InvestMate 的信任体系需要让用户看到数据、依据、前提和不确定性。

## 2. 信任来源

### 2.1 数据来源

页面需要展示：

- 数据源；
- 数据日期；
- 同步状态；
- 缺失字段提示。

### 2.2 计算依据

结构化分析需要展示：

- 输入持仓；
- 主要计算字段；
- 分布结果；
- 集中度结果。

### 2.3 AI 解释边界

AI 解释必须基于结构化结果，不直接生成无依据判断。

### 2.4 历史可回看

每次分析需要保存：

- 用户输入；
- 结构化结果；
- 中文解释；
- 数据日期；
- 生成时间。

## 3. 页面要求

报告页必须展示：

- 数据日期；
- 生成时间；
- 主要依据；
- 用户须知；
- 反馈入口。

## 4. 解释要求

每个重要结论尽量包含：

```text
结论
依据
需要关注的问题
```

## 5. 验收标准

- 用户能看到报告依据；
- 用户能看到数据日期；
- 用户能回看历史；
- 用户能提交反馈；
- 解释不脱离结构化结果。
