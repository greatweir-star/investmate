# 迭代节奏

用于记录 MVP 阶段的版本节奏和发布规则。

## 1. 节奏

MVP 阶段采用两周一个小版本的节奏。

每个版本只解决少量关键问题，避免一次修改过大。

## 2. 每周固定动作

- 整理用户反馈；
- 更新问题清单；
- 明确下周优先级；
- 检查核心流程是否顺畅。

## 3. 每两周固定动作

- 发布一个小版本；
- 总结新增能力；
- 记录已知问题；
- 更新内测用户。

## 4. 优先级规则

优先处理：

1. 用户看不懂的问题；
2. 用户无法完成流程的问题；
3. 数据展示不清楚的问题；
4. 历史记录和复盘相关问题。

## 5. 暂缓事项

- 大规模增长；
- 复杂视觉设计；
- 多端适配；
- 非核心高级功能。
