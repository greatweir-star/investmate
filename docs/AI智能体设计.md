# AI 智能体设计

## 1. 设计原则

InvestMate 的 AI 不应该只是一个聊天机器人，而应该是一组面向投研辅助流程的专用智能体。

每个智能体都有明确职责、输入、输出和边界。

第一阶段不追求复杂多 Agent 协作，先用清晰的模块化设计，为后续扩展预留空间。

## 2. 智能体列表

MVP 阶段包含四个智能体：

1. 市场分析 Agent；
2. 标的分析 Agent；
3. 持仓分析 Agent；
4. 复盘 Agent。

## 3. 市场分析 Agent

### 3.1 职责

根据指数、成交、波动和市场宽度等数据，生成市场状态概览。

### 3.2 输入

- 指数日线数据；
- 市场成交额；
- 涨跌分布；
- 行业表现；
- 历史市场状态。

### 3.3 输出

- 市场状态标签；
- 市场温度分；
- 主要支撑因素；
- 主要风险因素；
- 中文摘要。

### 3.4 边界

不预测明天涨跌，不生成交易指令。

## 4. 标的分析 Agent

### 4.1 职责

围绕单个股票、ETF 或指数，生成结构化投研辅助分析。

### 4.2 输入

- 标的基础信息；
- 日线行情；
- 估值数据；
- 行业信息；
- 用户投资 DNA；
- 用户提出的问题。

### 4.3 输出

- 趋势评分；
- 估值评分；
- 流动性评分；
- 风险评分；
- 用户适配度；
- 关键因素；
- 风险因素；
- 中文解释。

### 4.4 边界

不使用“必然”“保证”“无脑”等绝对化语言。

## 5. 持仓分析 Agent

### 5.1 职责

根据用户录入的持仓，识别组合结构问题。

### 5.2 输入

- 用户持仓列表；
- 持仓比例；
- 成本价，可选；
- 标的行业；
- 标的波动；
- 用户投资 DNA。

### 5.3 输出

- 单一标的集中度；
- 行业集中度；
- 风格暴露；
- 波动风险；
- 用户适配度；
- 组合摘要；
- 风险提示。

### 5.4 边界

不替用户自动调仓，不直接生成交易指令。

## 6. 复盘 Agent

### 6.1 职责

追踪历史输出，帮助用户和系统复盘。

### 6.2 输入

- 历史分析记录；
- 后续行情表现；
- 当时的市场状态；
- 当时的用户输入。

### 6.3 输出

- 观点回看；
- 后续表现；
- 当时判断的有效和失效因素；
- 复盘摘要。

### 6.4 边界

复盘不是为了证明 AI 永远正确，而是为了建立透明、可学习的系统。

## 7. Agent 协作流程

```text
用户输入问题
  ↓
系统识别任务类型
  ↓
调用对应 Agent
  ↓
Agent 读取结构化数据
  ↓
生成结构化结果
  ↓
生成中文解释
  ↓
保存历史记录
```

## 8. Prompt 规范

所有 Agent Prompt 必须包含：

- 角色定义；
- 输入字段；
- 输出 JSON Schema；
- 禁止表达；
- 合规提醒；
- 示例输出。

## 9. 后续扩展 Agent

未来可以增加：

- 新闻解释 Agent；
- 公告解读 Agent；
- 行业轮动 Agent；
- 财报解读 Agent；
- 用户行为分析 Agent；
- 内容生成 Agent。

但第一阶段只做最小闭环，避免 Agent 数量过多导致系统复杂度上升。
