# 报告页设计

## 1. 页面目标

报告页是 InvestMate 第一阶段最重要的结果页。

它需要让用户快速理解自己的组合结构，并愿意保存、回看和继续使用。

## 2. 路由建议

```text
/reports/{report_id}
```

MVP 阶段也可以先复用历史详情页。

## 3. 页面模块

### 3.1 报告头部

展示：

- 报告名称；
- 生成时间；
- 数据日期；
- 持仓数量。

### 3.2 组合摘要

展示一句话摘要：

```text
当前组合整体较分散，但需要关注权益类资产占比和部分方向重叠。
```

### 3.3 健康度卡片

展示：

- 组合健康度；
- 用户匹配度；
- 数据完整度。

### 3.4 集中度分析

展示：

- 最大单一持仓比例；
- 前三大持仓比例；
- 主要集中方向。

### 3.5 分布分析

展示：

- 行业分布；
- 资产类型分布；
- 风格分布。

### 3.6 重复暴露提示

展示可能重复的资产方向，例如多个宽基产品、多个同主题产品。

### 3.7 关注事项

展示 3 到 5 个需要用户进一步关注的问题。

### 3.8 中文解释

用自然语言解释结构化结果，但不替用户做最终判断。

### 3.9 行动区

按钮：

- 保存报告；
- 查看历史；
- 下次复盘提醒；
- 提交反馈。

## 4. 空状态

如果持仓数量不足：

```text
请至少录入 2 个标的，以便生成组合结构报告。
```

## 5. 数据不足状态

如果部分标的信息缺失：

```text
部分标的信息暂不完整，报告可能只展示基础结构。
```

## 6. 验收标准

- 页面可以展示报告摘要；
- 页面可以展示结构化模块；
- 页面可以保存报告；
- 页面可以进入历史记录；
- 页面可以提交反馈；
- 页面展示数据日期和生成时间。
