# 用户记忆系统

## 1. 目标

用户记忆系统用于让 InvestMate 从一次性报告工具，逐步演进为长期复盘产品。

记忆不是为了替用户做决定，而是为了让后续报告更贴近用户的历史、偏好和理解方式。

## 2. 记忆类型

第一阶段建议记录五类记忆：

1. 用户投资 DNA；
2. 历史报告；
3. 用户反馈；
4. 历史关注事项；
5. 解释偏好。

## 3. 用户投资 DNA

字段：

- risk_level；
- investment_horizon；
- asset_preferences；
- experience_level；
- max_drawdown_tolerance。

用途：

- 判断报告解释深度；
- 计算偏好匹配度；
- 生成更贴近用户场景的复盘重点。

## 4. 历史报告

保存每次报告的：

- report_id；
- report_type；
- positions；
- score_items；
- exposure_items；
- watch_items；
- explanation；
- data_date；
- generated_at。

用途：

- 支持历史回看；
- 支持周度复盘；
- 支持报告变化对比。

## 5. 用户反馈

保存：

- feedback_type；
- feedback_text；
- report_id；
- created_at。

用途：

- 判断哪些解释有帮助；
- 发现用户看不懂的模块；
- 优化下次报告展示顺序。

## 6. 历史关注事项

每次报告中的 watch_items 可以沉淀为历史关注事项。

字段：

- watch_item_id；
- title；
- description；
- first_seen_report_id；
- last_seen_report_id；
- status。

状态：

- active；
- improved；
- resolved；
- ignored。

## 7. 解释偏好

根据用户行为逐步推断：

- 喜欢简短摘要；
- 喜欢详细解释；
- 更关注集中度；
- 更关注分布；
- 更关注复盘变化。

MVP 阶段不需要复杂算法，可以先人工规则记录。

## 8. 记忆使用规则

记忆只能用于：

- 个性化解释；
- 历史回看；
- 复盘对比；
- 页面排序；
- 用户体验优化。

不得用于：

- 替用户做最终决定；
- 输出确定性判断；
- 隐藏不确定性；
- 扩大产品边界。

## 9. MVP 实现建议

MVP 先实现：

- 用户投资 DNA；
- 报告历史；
- 反馈记录；
- watch_items 保存。

后续再实现解释偏好和自动复盘。

## 10. 验收标准

- 报告可以保存；
- 历史报告可以回看；
- 用户反馈可以保存；
- 用户投资 DNA 可以读取；
- 周度复盘可以使用历史报告。
