# V0.0.02 视觉与交互规划

## 1. 版本目标

V0.0.01 重点是把 AI Native 的报告对象、证据追溯、反馈、周度复盘和后端预留能力跑通。

V0.0.02 的重点是让产品从“可用”升级为“好看、好懂、愿意继续用”。

核心目标：

> 用户第一次打开 InvestMate，就能感受到这是一个克制、聪明、有记忆、有审美的 AI 持仓体检产品。

## 2. 设计参考提炼

### 2.1 Alice 参考方向

参考 Alice 的方向，不是照抄视觉，而是提炼以下设计语言：

- 人格化表达：产品不只是工具，而是有稳定语气和角色感；
- 记忆感：强调“越用越懂你”；
- 多 Agent 感：不同能力像不同角色协作；
- 故事化长页：每个模块都有标题、场景、解释和行动；
- 轻微反工具感：避免过度企业 SaaS 味道。

### 2.2 小红书参考方向

参考小红书内容消费体验，提炼以下方向：

- 封面感：首屏和报告页要像一张值得截图的封面；
- 笔记感：复杂信息拆成短段、卡片、标签和重点句；
- 瀑布流感：历史报告、示例报告、复盘记录可以卡片化；
- 种草感：让用户愿意保存、分享、复看；
- 移动优先：卡片、留白、标题和行动按钮适合手机阅读。

### 2.3 Aino 参考方向

参考 Aino 的方向，不照搬具体页面，而是提炼“北欧克制 + 高级作品集 + 强编号系统 + 品牌化报告”的设计语言。

可借鉴点：

- 高级留白：页面不堆信息，让关键结论有呼吸感；
- 强文字层级：用大标题、编号、短句建立专业感；
- 作品集结构：每份报告像一个 case，而不是一条后台记录；
- 编号系统：报告、复盘、标的分析都用统一编号管理；
- Grid / List 双视图：历史页既能卡片浏览，也能专业列表查看；
- 克制品牌感：有高级感，但不炫技、不像金融大屏。

## 3. V0.0.02 设计定位

设计关键词：

- 温柔理性；
- 克制高级；
- 报告感；
- 个人化；
- 有记忆；
- 可截图；
- 移动端友好；
- 档案感；
- 作品集感。

避免：

- 冷冰冰的后台系统；
- 纯数据大屏；
- 过度金融交易感；
- 过度花哨的霓虹科技风；
- 把用户吓到的风险警告；
- 直接照搬设计机构网站导致产品可用性下降。

## 4. 核心视觉方向

### 4.1 色彩

建议采用“暖白背景 + 深墨文字 + 松绿色主色 + 琥珀提示色”。

用途：

- 暖白：降低金融产品压迫感；
- 深墨：保证专业和阅读体验；
- 松绿：表达理性、稳定、长期；
- 琥珀：提示关注事项，但不制造恐慌。

### 4.2 字体和排版

排版需要像一份高质量研究简报，而不是后台表格。

要求：

- 大标题要有情绪；
- 小标题要直接回答用户问题；
- 数字要突出；
- 解释文字不超过 3 行一段；
- 重要结论用短句卡片化；
- 关键报告编号要有存在感；
- 页面顶部可以用大号编号建立档案感。

### 4.3 卡片

所有核心信息都用卡片承载：

- 报告摘要卡；
- 健康度卡；
- 关注事项卡；
- 证据卡；
- 历史报告卡；
- 复盘变化卡。

卡片风格：

- 大圆角；
- 柔和阴影；
- 轻边框；
- 高留白；
- 标签化信息。

### 4.4 编号系统

V0.0.02 增加统一编号系统，强化报告档案感。

建议规则：

```text
RPT-001：AI 持仓体检报告
REV-001：周度复盘报告
AST-001：标的分析报告
DNA-001：用户投资 DNA 更新
```

编号用途：

- 报告页封面；
- 历史报告卡；
- 复盘对比；
- 分享截图；
- 用户长期档案。

### 4.5 Grid / List 双视图

历史页支持两种浏览方式：

- Grid：默认卡片书架，适合普通用户回看；
- List：专业列表，适合重度用户快速检索。

切换方式应轻量，不影响移动端使用。

## 5. 页面改造重点

### 5.1 首页

首页不再是普通仪表盘。

首屏结构：

```text
一句强标题
  ↓
一句副标题
  ↓
主按钮：开始持仓体检
  ↓
示例报告封面卡
  ↓
最近报告卡片流
```

标题示例：

```text
给你的持仓做一次 AI 体检
```

副标题示例：

```text
看清组合结构、集中度、重复暴露和下次复盘重点。
```

Aino 风格补充：

- 首页首屏不堆多个功能模块；
- 用一个强标题和一张示例报告封面建立品牌感；
- 最近报告卡片带编号，例如 RPT-003；
- 页面更像一个高质感产品入口，而不是后台首页。

### 5.2 持仓录入页

要降低录入压力。

优化方向：

- 默认提供示例组合；
- 支持一键清空；
- 支持快速添加常见 ETF；
- 总权重实时提示；
- 输入错误时用温和文案提示；
- 按钮文案更像任务开始，而不是表单提交。

按钮文案：

```text
生成我的体检报告
```

### 5.3 报告页

报告页是 V0.0.02 的核心。

需要做到：

- 第一屏就能截图；
- 3 秒看懂结论；
- 30 秒看懂依据；
- 2 分钟完成完整阅读；
- 一键保存和生成复盘。

模块顺序：

```text
报告封面
  ↓
一句话摘要
  ↓
健康度和匹配度
  ↓
三张重点卡
  ↓
结构分布
  ↓
为什么这么说
  ↓
AI 解释
  ↓
反馈和复盘入口
```

Aino 风格补充：

- 第一屏像作品集 case 封面；
- 顶部展示报告编号，例如 RPT-001；
- 报告标题使用大字号；
- 日期、类型、数据日期用小号信息条展示；
- 三张重点洞察卡像作品集摘要，而不是后台指标块；
- 页面不需要一屏塞满图表，先让核心结论站住。

### 5.4 历史页

历史页不要只像数据库列表。

优化为“报告书架”：

- 每条历史是一张报告卡；
- 展示报告编号、报告类型、摘要、日期、健康度；
- 支持按体检报告、周度复盘、标的分析筛选；
- 支持 Grid / List 双视图；
- 支持卡片展开详情；
- 点击进入独立报告页。

历史页的信息气质应接近“个人投资档案馆”，而不是后台表格。

### 5.5 设置页

设置页重点改为“我的投资 DNA”。

模块：

- 风险偏好；
- 投资期限；
- 偏好资产；
- 解释风格；
- 复盘频率；
- 数据和隐私说明。

## 6. 交互优化重点

### 6.1 生成过程要有仪式感

报告生成不应只是 loading。

建议展示阶段：

```text
正在整理持仓
  ↓
正在识别结构
  ↓
正在生成解释
  ↓
正在保存报告
```

### 6.2 证据展开

每个重要结论旁边提供：

```text
为什么这么说
```

点击后展开 evidence 卡片。

### 6.3 反馈轻量化

反馈入口不应像问卷。

建议用 3 个按钮：

- 有帮助；
- 看不懂；
- 想看更多。

再提供可选文字输入。

### 6.4 复盘引导

报告页底部增加：

```text
下次回来，我可以帮你对比这份报告。
```

按钮：

```text
生成周度复盘
```

### 6.5 移动端优先

所有核心流程必须在手机上可读、可点、可截图。

要求：

- 按钮高度不低于 44px；
- 重要卡片单列展示；
- 字号不低于 14px；
- 首屏不堆太多表格；
- 图表必须有文字解释。

### 6.6 视图切换

历史页增加 Grid / List 切换：

- Grid 默认展示，突出封面感和保存感；
- List 作为辅助，保留专业筛选效率；
- 移动端默认只展示 Grid，可把 List 放到更多菜单里。

## 8. 文案风格

V0.0.02 的文案需要更像“有审美的 AI 投资复盘空间”。

示例：

```text
不是让你马上行动，而是帮你看清结构。
```

```text
这份报告来自你当前录入的持仓比例。
```

```text
RPT-001 已保存。下次回来，我们可以一起对比它。
```

```text
这不是一条结论，而是一条可以追溯的观察。
```

## 9. 验收标准

V0.0.02 完成后应满足：

1. 首页首屏明确表达持仓体检；
2. 报告页第一屏可截图；
3. 报告页有明显高级感；
4. 报告页有统一编号系统；
5. 用户能展开查看证据；
6. 用户能轻量反馈；
7. 用户能进入周度复盘；
8. 历史页像报告书架，而不是后台表格；
9. 历史页支持 Grid / List 双视图；
10. 移动端阅读体验良好；
11. 所有文案克制、清晰、不绝对化；
12. 视觉风格统一，不像拼装页面。
