# 商业成功关键优化

## 1. 目标

本文档用于继续优化 InvestMate 从 MVP 到商业闭环之间最关键的十个方面。

当前产品规划已经支持 Codex 开始开发，但商业闭环还需要进一步聚焦。

## 2. 十个关键方面

### 2.1 目标用户聚焦

第一批用户不应泛化为所有投资者，而应聚焦为：有一定投资经验、持有股票或 ETF、经常被信息干扰、缺少复盘流程的个人用户。

### 2.2 核心付费理由

第一阶段不要先卖抽象的 AI 投资副驾驶，而是先验证用户是否愿意为“持仓结构体检”和“周期复盘”表达明确兴趣。

### 2.3 强钩子功能

最强入口应是 AI 持仓体检报告。

用户录入一组持仓后，系统生成结构化报告，帮助用户看清组合分布、集中度、重叠度和需要关注的问题。

### 2.4 信任体系

每个结论都需要展示依据，包括数据日期、计算口径、输入内容、结构化结果和解释来源。

### 2.5 产品边界

所有输出都应定位为信息整理和辅助理解，不替用户做最终判断。

### 2.6 获客路径

早期获客应围绕“持仓体检”内容展开，而不是泛泛宣传 AI 工具。

### 2.7 转化机制

转化点应放在个性化结果之后，例如保存报告、查看历史、订阅周期复盘、加入内测。

### 2.8 留存机制

留存核心是周期复盘。每周生成一次组合回看，比单次分析更容易形成使用习惯。

### 2.9 成本模型

需要控制免费次数、模型调用次数、报告生成成本和数据同步成本。

### 2.10 真实用户验证

必须用 20 到 50 名真实用户验证：是否愿意录入持仓、是否看得懂报告、是否愿意再次使用、是否愿意留下反馈。

## 3. 优先级

最高优先级：

1. AI 持仓体检报告；
2. 报告页；
3. 用户验证；
4. 留存复盘；
5. 成本模型。
