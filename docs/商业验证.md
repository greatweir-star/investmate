# 商业验证

## 1. 目标

验证 InvestMate 是否具备真实付费价值。

第一阶段不急于做复杂商业系统，而是验证用户是否愿意持续使用、录入信息、保存历史记录，并为个性化分析付费。

## 2. 用户价值假设

InvestMate 的价值来自四点：

1. 节省研究时间；
2. 降低信息整理成本；
3. 提供个性化持仓视角；
4. 建立长期复盘习惯。

## 3. 免费体验

免费体验用于获客。

能力：

- 基础市场状态；
- 有限次数标的分析；
- 简单历史记录。

目标：让用户理解产品价值。

## 4. 付费能力

付费能力优先放在个性化模块：

- 持仓分析；
- 用户投资 DNA；
- 长期历史记录；
- 周度复盘；
- 多关注列表。

## 5. 验证指标

重点指标：

- 用户是否完成首次分析；
- 用户是否录入持仓；
- 用户是否在 7 天内再次使用；
- 用户是否查看历史记录；
- 用户是否表示愿意付费。

## 6. 第一批商业实验

### 实验一：免费标的分析

目标：验证用户是否愿意输入标的并阅读完整结果。

### 实验二：持仓分析预约

目标：验证用户是否愿意提供自己的组合信息。

### 实验三：周度复盘报告

目标：验证用户是否愿意订阅周期性分析。

## 7. 判断标准

如果用户只是看首页，但不输入标的、不保存记录、不录入持仓，说明产品价值还不够强。

如果用户愿意录入持仓并反复查看历史记录，说明个性化方向成立。
