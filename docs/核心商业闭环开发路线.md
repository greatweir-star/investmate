# 核心商业闭环开发路线

## 1. 目标

本文档用于指导 Codex 和后续开发优先实现最可能形成商业闭环的产品路径。

InvestMate 第一阶段不应只做功能集合，而应优先打通：

```text
持仓录入
  ↓
AI 持仓体检报告
  ↓
报告保存
  ↓
历史回看
  ↓
周度复盘
  ↓
反馈与内测转化
```

## 2. 为什么优先做持仓体检报告

相比单个标的分析，持仓体检报告更接近用户真实痛点：

- 用户关心自己的组合是否清晰；
- 用户愿意为个人化结果投入更多注意力；
- 报告页更容易截图、分享和复盘；
- 周度复盘可以自然形成留存。

## 3. 第一优先级开发对象

### 3.1 持仓录入

用户可以手动录入资产代码、名称、比例和备注。

### 3.2 组合体检报告

系统根据录入内容生成结构化报告。

报告至少包含：

- 组合摘要；
- 健康度；
- 分散度；
- 分布情况；
- 关注事项；
- 中文解释；
- 数据日期。

### 3.3 报告保存

每次报告都需要保存到历史记录。

### 3.4 历史回看

用户可以回看过去生成的报告。

### 3.5 反馈入口

报告页需要提供反馈入口，用于收集用户是否看懂、是否有帮助、是否愿意继续使用。

## 4. 暂缓对象

以下内容不作为第一优先级：

- 复杂行情；
- 自动导入账户；
- 多端适配；
- 社区；
- 复杂付费系统；
- 过度视觉包装。

## 5. 开发顺序建议

```text
1. 完成后端 mock 报告接口
2. 完成持仓录入页面
3. 完成报告页
4. 完成报告历史保存
5. 完成历史详情页
6. 完成反馈入口
7. 完成周度复盘入口
```

## 6. 验收标准

- 用户可以录入持仓；
- 用户可以生成报告；
- 报告结构清晰；
- 报告可以保存；
- 历史记录可以回看；
- 用户可以提交反馈；
- 页面展示数据日期和说明。
