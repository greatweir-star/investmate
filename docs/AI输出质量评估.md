# AI 输出质量评估

## 1. 目标

AI 输出质量评估用于判断 InvestMate 的报告和解释是否可靠、清晰、可用。

质量评估不是只看文案是否流畅，而是检查输出是否结构化、可追溯、符合产品边界，并能帮助用户理解问题。

## 2. 评估对象

需要评估的对象包括：

- AI 持仓体检报告；
- 报告中文解释；
- 关注事项；
- 周度复盘；
- 用户反馈回复；
- 历史对比说明。

## 3. 质量维度

### 3.1 结构完整性

检查输出是否包含必要字段：

- summary；
- score_items；
- exposure_items；
- watch_items；
- explanation；
- data_date；
- evidence。

### 3.2 依据一致性

检查自然语言解释是否和结构化结果一致。

不允许解释中出现结构化结果没有支持的内容。

### 3.3 可读性

检查用户是否能快速理解：

- 当前报告说了什么；
- 哪些地方需要关注；
- 为什么这样解释；
- 数据是否完整。

### 3.4 边界合规性

检查输出是否符合产品边界：

- 不替用户做最终判断；
- 不制造紧迫感；
- 不夸大确定性；
- 不省略数据不足说明。

### 3.5 个性化匹配

检查解释是否结合用户投资 DNA 和历史反馈。

MVP 阶段至少应结合：

- risk_level；
- investment_horizon；
- asset_preferences；
- experience_level。

### 3.6 复盘连续性

检查周度复盘是否延续历史上下文：

- 是否引用上次报告；
- 是否对比变化；
- 是否延续上次关注事项；
- 是否保存本次结果。

## 4. 评分方式

每次 AI 输出可以按 0 到 5 分评估：

- 0：不可用；
- 1：字段严重缺失；
- 2：能读但不完整；
- 3：基本可用；
- 4：清晰且有依据；
- 5：清晰、有依据、可复盘、符合边界。

## 5. 自动检查项

后端可以先实现基础自动检查：

- 必填字段是否存在；
- data_date 是否存在；
- evidence 是否存在；
- explanation 是否为空；
- confidence 是否在合理范围；
- watch_items 是否为空。

## 6. 人工评估项

内测阶段每周抽样评估：

- 用户是否看懂；
- 解释是否过度抽象；
- 关注事项是否有帮助；
- 是否存在误导性表达；
- 是否需要调整 Prompt。

## 7. 用户反馈接入

用户反馈需要映射到质量问题：

- 看不懂：可读性问题；
- 没帮助：价值表达问题；
- 不准确：依据一致性问题；
- 想看更多：信息深度问题；
- 太复杂：解释层级问题。

## 8. 质量记录字段

建议保存：

- output_id；
- report_id；
- quality_score；
- missing_fields；
- boundary_check_status；
- user_feedback_summary；
- evaluated_at。

## 9. Prompt 迭代规则

当连续出现以下情况时，需要调整 Prompt：

- 用户多次反馈看不懂；
- 解释重复空泛；
- 输出缺少依据；
- 边界检查多次失败；
- 复盘没有引用历史。

## 10. 验收标准

- 每次输出可以自动检查字段完整性；
- 每次报告可以记录质量状态；
- 用户反馈可以归因到质量维度；
- Prompt 迭代有依据；
- 质量评估能服务产品迭代。
