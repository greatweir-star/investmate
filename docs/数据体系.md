# 数据体系

## 1. 数据体系定位

InvestMate 第一阶段的数据体系只服务于中低频投研辅助，不服务于盘中高频交易。

因此，系统优先接入稳定、结构化、可复盘的数据，而不是追求最低延迟。

第一阶段默认数据源为 Tushare Pro。

## 2. 数据使用原则

1. 数据必须可追溯。
2. 数据必须有更新时间。
3. 数据必须与分析结果绑定。
4. 数据缺失时必须明确提示。
5. 不使用来源不明的数据生成核心判断。

## 3. 第一阶段数据范围

### 3.1 股票基础信息

用途：

- 标的识别；
- 股票池过滤；
- 上市时间判断；
- ST 或退市风险过滤。

字段示例：

- ts_code；
- symbol；
- name；
- area；
- industry；
- market；
- list_date。

### 3.2 日线行情

用途：

- 趋势分析；
- 波动率计算；
- 回撤计算；
- 成交额分析。

字段示例：

- open；
- high；
- low；
- close；
- pre_close；
- change；
- pct_chg；
- vol；
- amount。

### 3.3 复权因子

用途：

- 计算复权价格；
- 避免分红送转造成指标失真。

### 3.4 每日基本面

用途：

- 估值分析；
- 市值分析；
- 换手率分析。

字段示例：

- total_mv；
- circ_mv；
- pe；
- pb；
- ps；
- turnover_rate。

### 3.5 指数行情

用途：

- 市场状态判断；
- 宽基指数对比；
- 风格环境判断。

优先指数：

- 上证指数；
- 深证成指；
- 沪深300；
- 中证500；
- 中证1000；
- 创业板指；
- 科创50。

### 3.6 ETF 行情

用途：

- ETF 分析；
- 用户组合中的宽基配置分析；
- 降低个股推荐色彩。

### 3.7 财务指标

用途：

- 基本面辅助分析；
- 过滤明显财务风险；
- 长期视角解释。

第一阶段只做基础展示，不做复杂财务模型。

## 4. 数据更新频率

### 每日更新

- 日线行情；
- 指数行情；
- ETF 行情；
- 每日基本面；
- 复权因子。

### 每周或按需更新

- 股票基础信息；
- 行业分类；
- 财务指标。

## 5. 数据处理流程

```text
Tushare API
  ↓
原始数据表
  ↓
清洗与标准化
  ↓
指标计算
  ↓
信号表
  ↓
决策引擎调用
```

## 6. 数据表分层

### 6.1 Raw Layer

保存原始数据，尽量不做修改。

表名示例：

- raw_stock_basic；
- raw_daily_price；
- raw_index_daily；
- raw_daily_basic。

### 6.2 Clean Layer

保存清洗后的标准化数据。

表名示例：

- asset_basic；
- asset_daily_price；
- index_daily_price。

### 6.3 Feature Layer

保存计算后的指标和信号。

表名示例：

- asset_signal_daily；
- market_signal_daily；
- portfolio_snapshot。

### 6.4 Output Layer

保存 AI 输出和用户可见结果。

表名示例：

- analysis_record；
- decision_output；
- user_report。

## 7. 数据质量检查

每次更新后，需要检查：

- 是否为交易日；
- 数据行数是否异常；
- 核心指数是否缺失；
- 价格字段是否为空；
- 成交额是否异常；
- 复权因子是否缺失。

如果数据异常，前端必须展示数据状态，不允许直接生成核心分析。

## 8. 第一阶段不接入的数据

暂不接入：

- Level-2；
- Tick；
- 实时盘口；
- 券商交易数据；
- 用户真实账户数据；
- 未授权新闻数据；
- 社交媒体情绪数据。

## 9. 后续扩展方向

未来可增加：

- 公告；
- 研报；
- 宏观数据；
- 利率数据；
- 北向资金；
- 融资融券；
- 行业景气度；
- 用户真实交易记录导入。

所有新增数据源都必须遵循可追溯、可复盘和合规使用原则。
