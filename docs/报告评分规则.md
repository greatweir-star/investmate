# 报告评分规则

## 1. 目标

报告评分规则用于支持 AI 持仓体检报告。

评分不是为了给用户结论，而是帮助用户理解组合结构的相对状态。

## 2. 总分结构

组合健康度建议使用 0 到 100 分。

第一版由五个部分组成：

- 分散度：25 分；
- 标的重叠度：20 分；
- 资产结构清晰度：20 分；
- 用户偏好匹配度：20 分；
- 数据完整度：15 分。

## 3. 分散度

观察组合是否过度集中在少数标的。

参考字段：

- 最大单一持仓比例；
- 前三大持仓比例；
- 持仓数量。

MVP 可用简单规则：

- 最大单一持仓低于 20%，分散度较好；
- 最大单一持仓 20% 到 40%，需要关注；
- 最大单一持仓高于 40%，集中度较高。

## 4. 标的重叠度

观察多个标的是否暴露在相似方向。

MVP 可先基于 asset_type、industry、style 做粗粒度判断。

后续再接入更细的成分股重叠或主题重叠。

## 5. 资产结构清晰度

观察组合中不同类型资产是否清晰。

第一版可以使用：

- 股票；
- ETF；
- 指数；
- 现金或其他。

## 6. 用户偏好匹配度

基于用户投资 DNA 判断组合是否与用户偏好相符。

参考字段：

- risk_level；
- investment_horizon；
- asset_preferences；
- experience_level。

## 7. 数据完整度

观察报告生成所需字段是否完整。

如果部分标的缺少行业、类型或最新数据，报告需要降低完整度说明。

## 8. 输出要求

报告不只展示分数，还必须展示每个分数的解释。

格式：

```text
指标名称
得分
说明
依据
```

## 9. 验收标准

- 能生成总分；
- 能生成分项分数；
- 每个分数有中文解释；
- 数据不足时有提示；
- 分数不作为最终判断。
