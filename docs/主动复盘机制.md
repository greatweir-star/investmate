# 主动复盘机制

## 1. 目标

主动复盘机制用于让 InvestMate 从用户主动点击工具，逐步演进为能提醒用户回看历史和检查变化的 AI 产品。

第一阶段不做复杂通知系统，但需要在产品设计和数据结构中预留主动复盘能力。

## 2. 主动复盘的定义

主动复盘不是替用户做决定，而是提醒用户：

- 上次报告中的关注事项是否仍然存在；
- 组合结构是否有明显变化；
- 用户是否需要回看历史记录；
- 是否到了周期性复盘时间。

## 3. 触发条件

### 3.1 MVP 手动触发

MVP 阶段先支持用户手动点击：

```text
生成周度复盘
```

### 3.2 定时触发

后续支持每周固定时间生成复盘提醒。

### 3.3 数据变化触发

当数据源更新后，如果用户有历史报告，可以提示用户重新查看。

### 3.4 用户行为触发

当用户多次查看同一报告或同一类关注事项，可以提示用户生成复盘。

## 4. 复盘内容

主动复盘报告应包含：

1. 本次摘要；
2. 与上次报告相比的变化；
3. 仍然存在的关注事项；
4. 新出现的关注事项；
5. 用户上次反馈的延续；
6. 下次建议回看的时间。

## 5. 工作流

```text
读取用户最新持仓
  ↓
读取最近一次报告
  ↓
读取历史关注事项
  ↓
读取用户反馈
  ↓
生成变化摘要
  ↓
生成复盘报告
  ↓
保存到历史记录
```

## 6. 页面入口

需要在以下位置提供入口：

- 报告详情页；
- 历史记录页；
- 首页最近记录区；
- 设置页复盘偏好。

## 7. 提醒文案

示例：

```text
距离上次体检已经过去一周，可以生成一次组合复盘。
```

```text
上次报告中的关注事项仍可继续回看。
```

## 8. 用户控制

用户应可以控制：

- 是否显示复盘提醒；
- 复盘周期；
- 是否保存复盘历史；
- 是否清除历史记录。

## 9. MVP 范围

MVP 只实现：

- 手动生成周度复盘；
- 基于最近一次报告做简单对比；
- 保存复盘结果；
- 在历史记录中展示。

暂不实现：

- 推送通知；
- 邮件提醒；
- 多渠道消息；
- 自动生成并发送。

## 10. 验收标准

- 用户可以从报告详情进入复盘；
- 系统可以读取最近报告；
- 系统可以生成变化摘要；
- 复盘结果可以保存；
- 用户可以在历史中回看复盘。
