# 主动复盘机制

## 1. 目标

主动复盘机制用于让 InvestMate 从用户主动点击工具，逐步演进为能提醒用户回看历史和检查变化的 AI 产品。

第一阶段不做复杂通知系统，但需要在产品设计和数据结构中预留主动复盘能力。

## 2. 主动复盘的定义

主动复盘不是替用户做决定，而是提醒用户：

- 上次报告中的关注事项是否仍然存在；
- 组合结构是否有明显变化；
- 用户是否需要回看历史记录；
- 是否到了周期性复盘时间。

## 3. 触发条件

MVP 可支持手动触发。

后续支持以下触发条件：

### 3.1 时间触发

例如每周一次、每月一次。

### 3.2 用户行为触发

例如用户更新持仓后，提示重新生成报告。

### 3.3 数据变化触发

例如数据日期更新后，提示可以重新查看组合结构。

### 3.4 历史关注事项触发

例如上次报告中存在 active watch_items，本次提醒继续回看。

## 4. 主动复盘输出

主动复盘输出包括：

- review_id；
- user_id；
- current_report_id；
- previous_report_id；
- changes；
- continued_watch_items；
- resolved_items；
- next_review_focus；
- generated_at。

## 5. 复盘页面模块

页面展示：

1. 本次复盘摘要；
2. 与上次相比的变化；
3. 仍需关注的问题；
4. 已改善的问题；
5. 下次复盘重点；
6. 用户反馈入口。

## 6. MVP 实现

MVP 阶段实现：

- 用户在历史记录页手动点击“生成复盘”；
- 系统选择最近两份报告；
- 生成对比摘要；
- 保存复盘记录。

## 7. 后续增强

后续增强：

- 周度自动提醒；
- 邮件或消息提醒；
- 数据更新后提醒；
- 关注事项状态跟踪；
- 用户偏好驱动提醒频率。

## 8. 边界要求

主动复盘只能提醒用户回看和理解变化，不能替用户做最终判断。

所有复盘输出必须包含数据日期、生成时间和解释边界。

## 9. 验收标准

- 可以从历史报告生成复盘；
- 可以展示变化摘要；
- 可以展示持续关注事项；
- 可以保存复盘记录；
- 可以提交反馈。
