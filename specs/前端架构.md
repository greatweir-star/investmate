# 前端架构

## 1. 技术选型

MVP 前端建议使用：

- Next.js；
- React；
- TypeScript；
- Tailwind CSS。

## 2. 目录结构

```text
frontend/
├── app/
│   ├── page.tsx
│   ├── assets/page.tsx
│   ├── portfolio/page.tsx
│   ├── history/page.tsx
│   └── settings/page.tsx
├── components/
│   ├── layout/
│   ├── cards/
│   ├── forms/
│   └── analysis/
├── lib/
│   ├── api.ts
│   ├── types.ts
│   └── format.ts
└── styles/
```

## 3. 页面路由

### `/`

首页。

展示：

- 市场状态；
- 风险摘要；
- 快速分析入口；
- 最近记录。

### `/assets`

标的分析页。

展示：

- 搜索框；
- 标的基础信息；
- 分析评分；
- 关键因素；
- 风险因素；
- 中文解释。

### `/portfolio`

持仓分析页。

展示：

- 持仓录入；
- 组合结构；
- 风险摘要；
- 用户匹配度。

### `/history`

历史记录页。

展示：

- 记录列表；
- 详情入口。

### `/settings`

设置页。

展示：

- 数据源配置；
- AI 服务配置；
- 用户投资 DNA。

## 4. 核心组件

### MarketStatusCard

输入：MarketStatus。

展示市场状态、评分、风险等级和摘要。

### AnalysisScorePanel

输入：scores。

展示趋势、估值、流动性、风险、用户匹配度。

### FactorList

输入：key_factors 和 risk_factors。

分组展示关键因素和风险因素。

### ExplanationPanel

输入：explanation。

展示中文解释。

### PositionEditor

输入和编辑持仓列表。

### HistoryList

展示历史分析记录。

## 5. API 调用规范

前端统一通过 `lib/api.ts` 调用后端。

不要在页面组件中直接写 fetch 逻辑。

API 返回必须按以下结构处理：

```ts
type ApiResponse<T> = {
  success: boolean;
  data: T | null;
  error: null | {
    code: string;
    message: string;
  };
  meta: {
    request_id?: string;
    generated_at?: string;
    data_date?: string;
    version?: string;
  };
};
```

## 6. 状态管理

MVP 不需要引入复杂状态管理库。

优先使用：

- React state；
- URL params；
- 服务端返回数据。

## 7. UI 原则

1. 摘要优先；
2. 风险提示不能隐藏；
3. 数据日期必须可见；
4. 图表可以后置；
5. 用户不需要理解全部指标也能看懂页面。

## 8. MVP 验收

前端完成后应能：

- 访问首页；
- 输入标的并请求分析接口；
- 录入持仓并请求分析接口；
- 查看历史记录；
- 修改基础设置。
