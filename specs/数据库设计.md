# 数据库设计

## 1. 设计目标

数据库第一阶段服务于以下能力：

1. 保存 Tushare 数据；
2. 保存用户投资 DNA；
3. 保存用户持仓；
4. 保存系统分析结果；
5. 支持历史复盘。

推荐使用 PostgreSQL。

## 2. 命名规范

- 表名使用英文小写加下划线；
- 字段名使用英文小写加下划线；
- 文档说明使用中文；
- 时间字段统一使用 `created_at`、`updated_at`；
- 所有分析结果保留 `data_date`。

## 3. 用户表

### users

用途：保存用户基础信息。

字段：

```text
id                  uuid 主键
email               text 可选
nickname            text 可选
created_at          timestamp
updated_at          timestamp
```

## 4. 用户投资 DNA 表

### user_investment_dna

用途：保存用户风险偏好和投资画像。

字段：

```text
id                          uuid 主键
user_id                     uuid
risk_level                  text
investment_horizon          text
max_drawdown_tolerance      numeric
asset_preferences           jsonb
style_preference            text
trading_frequency           text
experience_level            text
behavior_tags               jsonb
created_at                  timestamp
updated_at                  timestamp
```

用途：保存股票和 ETF 日线数据。

字段：

```text
close               numeric
pre_close           numeric
change_amount       numeric
pct_chg             numeric
```


## 13. 分析记录表

### analysis_record

用途：保存每次 AI 分析输出。

字段：

```text
id                  uuid 主键
user_id             uuid 可选
analysis_type       text
input_payload       jsonb
structured_result   jsonb
natural_language    text
data_date           date
prompt_version      text
model_version       text
created_at          timestamp
```

analysis_type 示例：

- market_status；
- asset_review；
- portfolio_review；
- history_review。

## 14. 数据同步日志表

用途：记录数据同步任务。

字段：

```text
id                  uuid 主键
source              text
data_type           text
trade_date          date
status              text
row_count           integer
error_message       text
started_at          timestamp
finished_at         timestamp
```
