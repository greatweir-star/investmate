# 开发路线图

本文档定义 InvestMate 从 0 到 1 的开发阶段。当前阶段适合 OPC（一人公司）快速验证，不追求大而全。

## 总原则

先验证价值，再完善系统。

第一阶段不追求完整金融平台，而是验证用户是否愿意使用 AI 投研辅助系统来帮助自己做投资判断。

## Phase 0：仓库初始化

目标：建立可交给 Codex 开发的中文 AI Native 产品仓库。

交付物：

- README.md
- 项目愿景.md
- 产品原则.md
- 开发路线图.md
- REPOSITORY_SPEC.md
- docs/ 产品文档
- specs/ 技术规格
- prompts/ Prompt 规范
- tasks/ 开发任务

完成标准：

- Codex 打开仓库后知道产品定位；
- Codex 知道第一阶段做什么、不做什么；
- 关键模块有输入、输出和边界定义。

## Phase 1：投研辅助 MVP

目标：跑通从数据到 AI 解释再到用户界面的最小闭环。

核心功能：

1. 接入 Tushare 基础数据；
2. 建立本地数据库；
3. 计算市场状态指标；
4. 计算标的基础分析指标；
5. 支持用户手动录入持仓；
6. 生成结构化 AI 辅助分析结果；
7. 前端展示首页、标的分析页、持仓页、历史页。

建议技术栈：

- 前端：Next.js / React；
- 后端：FastAPI 或 Next.js API Routes；
- 数据库：PostgreSQL；
- 数据源：Tushare Pro；
- AI：LLM + 结构化 Prompt；
- 部署：Vercel + 云数据库，或 Docker 单机部署。

完成标准：

- 可以每天更新 A 股基础行情；
- 可以输入一个标的并生成结构化分析；
- 可以录入持仓并生成组合风险摘要；
- 所有 AI 输出均有结构化结果和中文解释；
- 历史输出可保存。

## Phase 2：用户投资 DNA

目标：让系统开始理解用户，而不是只分析市场。

核心功能：

1. 用户风险问卷；
2. 投资期限设置；
3. 最大回撤偏好；
4. 资产偏好；
5. 交易频率偏好；
6. 投资目标记录；
7. 用户画像版本化。

完成标准：

- 不同风险偏好的用户看到不同的辅助分析侧重点；
- 系统输出能解释“为什么这个提示适合你”；
- 用户画像可以被后续 Agent 调用。

## Phase 3：复盘与信任系统

目标：建立长期可信度，而不是只生成一次性内容。

核心功能：

1. 历史输出记录；
2. 输出版本管理；
3. 观点回看；
4. 标的后续表现追踪；
5. 每周复盘摘要；
6. 错误案例记录。

完成标准：

- 用户可以查看系统过去说过什么；
- 系统可以解释后续市场变化与原判断之间的关系；
- 产品开始积累可复盘的决策历史资产。

## Phase 4：产品化与商业验证

目标：从工具验证转向付费验证。

可能方向：

1. 免费版：基础市场和标的分析；
2. 付费版：持仓分析、用户投资 DNA、历史复盘；
3. 高级版：个性化研究模板、组合风险报告、定期邮件；
4. B 端：投研内容生成 API 或内部研究助手。

验证指标：

- 用户是否重复使用；
- 用户是否愿意录入持仓；
- 用户是否愿意保存历史输出；
- 用户是否愿意为个性化分析付费。

## Phase 5：自动化投研工作流

目标：让 AI 主动完成更多日常投研任务。

可能功能：

1. 每日市场状态更新；
2. 用户关注列表自动更新；
3. 持仓风险变化提醒；
4. 行业轮动摘要；
5. 组合月度复盘；
6. 多 Agent 协作。

注意：即便进入 Phase 5，系统仍然保持“投资辅助”定位，不进入自动交易。

## 当前优先级

当前只做 Phase 0 和 Phase 1。

任何不服务于 Phase 1 的功能，都暂缓。
