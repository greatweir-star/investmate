# 页面规格：持仓分析

## 1. 页面目标

持仓分析页用于帮助用户理解当前组合结构中的主要问题。

页面回答的问题是：

> 我的组合结构是否存在明显集中、偏离或风险暴露？

## 2. 路由

```text
/portfolio
```

## 3. 页面组成

页面包含以下组件：

1. AppHeader；
2. PositionEditor；
3. PortfolioSummaryCard；
4. PortfolioRiskPanel；
5. ExposureBreakdown；
6. ExplanationPanel；
7. SavePortfolioButton。

## 4. 数据依赖

### 4.1 保存持仓

接口：

```text
POST /api/portfolio
```

### 4.2 分析持仓

接口：

```text
POST /api/analysis/portfolio
```

## 5. 用户流程

```text
进入页面
  ↓
录入持仓
  ↓
保存持仓
  ↓
点击分析
  ↓
系统生成组合结构摘要
  ↓
查看风险因素和解释
  ↓
结果写入历史记录
```

## 6. 输入字段

PositionEditor 支持字段：

- asset_code；
- asset_name；
- weight；
- cost_price；
- note。

其中 `asset_code` 和 `weight` 为必填。

## 7. 校验规则

1. 持仓比例必须大于 0；
2. 总比例可以小于 100%，代表保留现金或其他未录入资产；
3. 总比例超过 100% 时给出提示；
4. 标的不存在时提示用户检查输入；
5. 至少录入一个标的才可以分析。

## 8. 输出内容

页面展示：

- 组合摘要；
- 总体风险等级；
- 单一标的集中度；
- 行业分布；
- 风格暴露；
- 与用户投资 DNA 的匹配度；
- 风险因素；
- 中文解释。

## 9. 空状态

首次进入：

```text
请先录入你的持仓，系统会帮助你查看组合结构。
```

## 10. 异常状态

数据不足：

```text
部分标的数据不足，分析结果可能不完整。
```

分析失败：

```text
持仓分析暂时不可用，请稍后重试。
```

## 11. 验收标准

1. 可以录入多条持仓；
2. 可以校验持仓比例；
3. 可以提交持仓分析；
4. 可以展示组合结构；
5. 可以展示中文解释；
6. 可以保存历史记录。
