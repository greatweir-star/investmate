# 页面规格：首页

## 1. 页面目标

首页是 InvestMate MVP 的默认入口。

它要回答用户的第一个问题：

> 当前市场环境和我的下一步研究重点是什么？

首页不做复杂投研展开，只做信息摘要和入口分发。

## 2. 路由

```text
/
```

## 3. 页面组成

首页包含以下组件：

1. AppHeader；
2. MarketStatusCard；
3. RiskSummaryCard；
4. QuickAnalyzeBox；
5. RecentHistoryList；
6. FooterNotice。

## 4. 页面数据依赖

### 4.1 市场状态

接口：

```http
GET /api/market/status
```

用于：

- MarketStatusCard；
- RiskSummaryCard。

### 4.2 最近记录

接口：

```http
GET /api/analysis/history?limit=5
```

用于：

- RecentHistoryList。

## 5. 页面布局

推荐布局：

```text
AppHeader
  ↓
页面标题区
  ↓
MarketStatusCard + RiskSummaryCard
  ↓
QuickAnalyzeBox
  ↓
RecentHistoryList
  ↓
FooterNotice
```

桌面端：卡片两列布局。

移动端：单列布局。

## 6. 空状态

如果市场数据不存在：

- MarketStatusCard 显示“暂无最新市场数据”；
- 引导用户前往设置页同步数据。

如果历史记录为空：

- RecentHistoryList 显示“暂无历史分析”；
- 引导用户进行第一次标的分析。

## 7. 异常状态

### 7.1 API 请求失败

显示：

```text
数据暂时无法加载，请稍后重试。
```

### 7.2 数据日期过旧

显示：

```text
当前数据不是最新交易日，请检查数据同步状态。
```

## 8. 用户交互

### 8.1 点击快速分析按钮

用户在 QuickAnalyzeBox 输入代码或名称后，点击按钮。

行为：

- 跳转到 `/assets?keyword=用户输入内容`。

### 8.2 点击历史记录

跳转到历史详情页。

MVP 可以先跳转到 `/history`。

## 9. 验收标准

首页完成后应满足：

1. 可以正常访问 `/`；
2. 可以展示市场状态 mock 数据；
3. 可以展示风险摘要；
4. 可以输入标的并跳转；
5. 可以展示最近历史记录；
6. 空状态和异常状态有中文提示。
