# 页面规格：标的分析

## 1. 页面目标

标的分析页用于帮助用户围绕一个股票、ETF 或指数形成结构化理解。

页面回答的问题是：

> 这个标的当前有哪些值得进一步关注的因素？

## 2. 路由

```text
/assets
```

## 3. 页面组成

页面包含以下组件：

1. AppHeader；
2. AssetSearchInput；
3. AssetBasicInfo；
4. AssetScorePanel；
5. FactorList；
6. ExplanationPanel；
7. DataFreshnessNotice。

## 4. 数据依赖

### 4.1 标的搜索

接口：

```text
GET /api/assets/search?keyword=xxx
```

### 4.2 标的分析

接口：

```text
POST /api/analysis/asset
```

## 5. 用户流程

```text
进入页面
  ↓
输入代码或名称
  ↓
选择标的
  ↓
点击分析
  ↓
查看结构化结果
  ↓
查看中文解释
  ↓
结果写入历史记录
```

## 6. 页面布局

推荐布局：

```text
页面标题
  ↓
AssetSearchInput
  ↓
AssetBasicInfo
  ↓
AssetScorePanel
  ↓
FactorList
  ↓
ExplanationPanel
  ↓
DataFreshnessNotice
```

## 7. 空状态

如果用户未输入：

```text
请输入股票、ETF 或指数代码开始分析。
```

如果未找到标的：

```text
未找到相关标的，请检查代码或名称。
```

如果数据不足：

```text
当前数据不足，暂时无法生成完整分析。
```

## 8. 异常状态

### API 请求失败

显示：

```text
分析服务暂时不可用，请稍后重试。
```

### 数据日期过旧

显示：

```text
当前数据不是最新交易日，请在设置页检查数据同步状态。
```

## 9. 验收标准

1. 可以输入标的并触发分析；
2. 可以展示基础信息；
3. 可以展示评分；
4. 可以展示关键因素和风险因素；
5. 可以展示中文解释；
6. 异常状态有中文提示；
7. 结果可以进入历史记录。
