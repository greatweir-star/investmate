# 页面规格：历史记录

## 1. 页面目标

历史记录页用于保存和回看系统过去生成的分析结果。

页面回答的问题是：

> 系统过去输出过什么，我能否回看当时的输入、结果和解释？

## 2. 路由

```text
/history
```

## 3. 页面组成

页面包含以下组件：

1. AppHeader；
2. HistoryFilter；
3. HistoryTable；
4. HistoryDetailDrawer；
5. EmptyState。

## 4. 数据依赖

### 4.1 获取历史列表

接口：

```text
GET /api/analysis/history
```

### 4.2 获取历史详情

接口：

```text
GET /api/analysis/{analysis_id}
```

## 5. 用户流程

```text
进入历史记录页
  ↓
查看历史列表
  ↓
筛选类型或时间
  ↓
点击某条记录
  ↓
查看完整详情
```

## 6. 历史列表字段

HistoryTable 展示字段：

- created_at；
- analysis_type；
- target_name；
- summary；
- risk_level；
- confidence；
- data_date。

## 7. 筛选条件

MVP 支持：

- 分析类型；
- 时间范围。

后续支持：

- 标的代码；
- 风险等级；
- 用户反馈。

## 8. 详情内容

HistoryDetailDrawer 展示：

- 用户原始输入；
- 结构化结果；
- 中文解释；
- 数据日期；
- Prompt 版本；
- 模型版本。

## 9. 空状态

没有历史记录时显示：

```text
暂无历史分析。完成一次标的分析或持仓分析后，记录会显示在这里。
```

## 10. 异常状态

请求失败时显示：

```text
历史记录暂时无法加载，请稍后重试。
```

## 11. 验收标准

1. 可以展示历史列表；
2. 可以查看详情；
3. 可以筛选分析类型；
4. 空状态有清晰提示；
5. 详情包含结构化结果和中文解释。
