# V0.0.02 视觉交互任务清单

## 1. 版本目标

V0.0.02 聚焦界面美学和交互逻辑。

目标不是新增复杂业务功能，而是把 V0.0.01 的 AI Native 闭环做得更好看、更好懂、更愿意用。

## 2. 必读文档

开发前阅读：

1. `docs/V0.0.02视觉与交互规划.md`
2. `docs/报告页设计.md`
3. `docs/报告字段规格.md`
4. `docs/解释可追溯设计.md`
5. `docs/主动复盘机制.md`
6. `docs/AI输出质量评估.md`
7. `docs/落地页转化文案.md`

## 3. Task 001：统一视觉基础

目标：建立统一视觉系统。

要求：

- 调整全局背景为暖白或浅米色；
- 建立主色、辅助色、提示色；
- 统一圆角、阴影、边框；
- 统一标题、正文、标签、数字样式；
- 清理不一致的按钮和卡片样式。

验收：

- 页面风格统一；
- 不像后台管理系统；
- 卡片和按钮有一致设计语言。

## 4. Task 002：首页首屏重做

目标：首页第一眼明确表达产品价值。

要求：

- 标题改为“给你的持仓做一次 AI 体检”；
- 副标题说明组合结构、集中度、重复暴露和复盘；
- 主按钮进入 `/portfolio`；
- 添加示例报告卡；
- 最近记录改为卡片流。

验收：

- 用户 5 秒内知道产品用途；
- 主按钮明确；
- 首屏有记忆点。

## 5. Task 003：持仓录入页优化

目标：降低录入压力，增强任务感。

要求：

- 增加总权重实时提示；
- 增加一键清空；
- 增加示例组合恢复；
- 错误提示更温和；
- loading 改为生成阶段提示。

生成阶段：

```text
正在整理持仓
正在识别结构
正在生成解释
正在保存报告
```

验收：

- 用户知道正在发生什么；
- 输入错误时不会挫败；
- 按钮文案更有行动感。

## 6. Task 004：独立报告页视觉升级

目标：把报告页做成 V0.0.02 的视觉核心。

要求：

- 新增或优化 `/reports/[id]`；
- 第一屏展示报告封面；
- 展示一句话摘要；
- 展示健康度和匹配度；
- 展示 3 张重点洞察卡；
- 展示结构分布；
- 展示 AI 解释；
- 展示反馈和复盘入口。

验收：

- 报告页第一屏可截图；
- 用户 3 秒内看懂结论；
- 页面有高级感。

## 7. Task 005：证据展开交互

目标：让解释可追溯。

要求：

- 每个重点洞察有“为什么这么说”；
- 点击后展开 evidence；
- 展示 source_field、source_value、description；
- 数据不足时展示说明。

验收：

- 用户能看到依据；
- evidence 不扰乱主阅读；
- 展开和收起顺滑。

## 8. Task 006：反馈交互优化

目标：让反馈变轻。

要求：

- 报告页底部增加 FeedbackChips；
- 三个快捷反馈：有帮助、看不懂、想看更多；
- 可选文本输入；
- 提交后展示温和确认文案。

验收：

- 用户 1 秒内知道怎么反馈；
- 反馈不打断阅读；
- 后端可以收到反馈。

## 9. Task 007：历史页改为报告书架

目标：让历史页从表格变成复盘空间。

要求：

- 历史记录以卡片展示；
- 每张卡展示摘要、类型、日期、健康度；
- 支持筛选；
- 点击进入报告详情；
- 保留结构化详情入口。

验收：

- 历史页不再像后台表；
- 用户愿意回看；
- 移动端单列体验良好。

## 10. Task 008：周度复盘 CTA

目标：强化长期使用感。

要求：

- 报告页底部添加 WeeklyReviewCTA；
- 文案强调“下次回来可以对比这份报告”；
- 可以触发 mock 周度复盘；
- 复盘结果进入历史。

验收：

- 用户理解复盘价值；
- 周度复盘入口清楚；
- 复盘结果可回看。

## 11. Task 009：移动端优化

目标：确保核心体验适合手机。

要求：

- 所有主按钮高度不低于 44px；
- 报告页卡片移动端单列；
- 表格改为卡片或可横滑；
- 首屏减少拥挤；
- 截图时标题和核心结论完整可见。

验收：

- 手机宽度下无明显溢出；
- 关键流程可单手完成；
- 报告页适合截图。

## 12. Task 010：文案统一

目标：让产品语气更像聪明的 AI 伙伴，而不是后台系统。

要求：

- 短句；
- 克制；
- 有温度；
- 不绝对化；
- 不制造交易冲动。

示例：

```text
不是让你马上行动，而是帮你看清结构。
```

```text
这条结论来自你当前录入的持仓比例。
```

验收：

- 所有页面中文文案统一；
- 没有生硬的系统提示；
- 没有夸张表达。

## 13. 完成定义

V0.0.02 完成后，需要实现：

1. 首页首屏有强钩子；
2. 报告页视觉明显升级；
3. 历史页卡片化；
4. 证据可展开；
5. 反馈更轻；
6. 复盘入口更自然；
7. 移动端体验良好；
8. 全站视觉统一。
